## はじめに

最近(今さらになって)、初めて Github Codespaces を使って Web ブラウザ上で開発しようとしたところ、数字や一部のキーが効かなくなる現象が発生しました。
この記事は、その原因と対応策を調査したときの備忘録です。

## 原因

結論として、使用している Web ブラウザの拡張機能 **Vimium**（ブラウザ操作を Vim のキーバインドで行える拡張機能）が、Codespaces 上のエディタとキーバインドで干渉していました。

最初は原因が分からなかったのですが、`f` キーを押したときに Vimium のリンクヒントモードが起動したことで察しました。

## 対策

### 一時的な対策（ページ単位で無効化）

1. ブラウザの拡張機能一覧から **Vimium** を選択
2. 「**Exclude Vimium keys on this page**」を選択
   ![](https://storage.googleapis.com/zenn-user-upload/1a2753a94c3a-20250811.png)
3. **Save Changes** を選択
   ![](https://storage.googleapis.com/zenn-user-upload/2ba47f115b23-20250811.png)
4. 設定を保存後、Codespaces に戻るとそのページでは Vimium が無効化され、全てのキー入力が可能になります。

### 恒久的な対策（URLパターンで無効化）

Codespaces の URL は新規作成時ごとに異なるため、毎回除外設定を行うのは手間です。
そのため、URL のパターンにマッチした場合に自動で無効化されるようにします。

1. 「**See all exclusion rules on the Options page**」の **Options** を選択
2. 「**Excluded URLs and keys**」の「**Add rules**」を選択して，以下のURLを入力する

   ```
   https://*.github.dev/
   ```

   ![](https://storage.googleapis.com/zenn-user-upload/105ef24399cf-20250811.png)
3. ページ下部の「**Save changes**」を選択

## まとめ

Github Codespaces は、軽微な修正や開発環境のセットアップ不要なハンズオンなどで非常に便利なサービスです。
一方、Vimium もキーボード操作を多用する開発者にとって強力なブラウザ操作ツールです。
しかし、Web 上のエディタを開くと Vimium が自動適用され、キー入力が制限されることがあります。
同じ状況になった場合は、URL 除外設定を活用して快適な開発環境を保つことができます。
